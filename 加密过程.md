下面用你的 SM2 代码为基础，构建一个完整的报关数据流转过程。

第一步：密钥初始化

企业 A:   keyPair_A = (sk_A, pk_A)     // SM2 密钥对
海关:     keyPair_Customs = (sk_C, pk_C) // SM2 密钥对
sk = 私钥（签名 + 解密），pk = 公钥（验签 + 加密）
每个企业的公钥 pk_A 在链上公开注册，海关的公钥 pk_C 也公开
第二步：企业构造报关数据包
企业端要提交的数据有三层保护：


# 原始报关数据
declaration = {
    "decl_id": "202605120001",
    "company": "ABC Corp",
    "goods": "电子元器件",
    "value": 500000,
    "hs_code": "8542.3100",
    "timestamp": "2026-05-12T10:00:00Z"
}
企业做如下操作：


① 对报关单整体做 SM3 摘要
   hash = SM3(declaration_json)

② 用企业私钥对摘要签名
   signature = SM2_Sign(sk_A, hash)

③ 对商业敏感字段（价值、品名等）用海关公钥加密
   encrypted_value = SM2_Encrypt(pk_C, "500000")
   encrypted_goods = SM2_Encrypt(pk_C, "电子元器件")

④ 组装提交数据包
   submit_packet = {
       "decl_id": "202605120001",
       "company": "ABC Corp",                    # 明文
       "hs_code": "8542.3100",                    # 明文
       "encrypted_goods": encrypted_goods,        # 密文
       "encrypted_value": encrypted_value,        # 密文
       "signature": signature,                     # 签名
       "public_key": pk_A,                        # 企业公钥（用于验签）
       "timestamp": "2026-05-12T10:00:00Z"
   }
此时数据包结构：


┌─────────────────────────────────────────────┐
│                 提交数据包                   │
│  ├─ 明文区：decl_id, company, hs_code        │
│  ├─ 密文区（海关公钥加密）：goods, value      │
│  │   → 只有海关私钥能解密                    |
│  └─ 签名区（企业私钥签名）：hash              │
│      → 任何人可用企业公钥验签                │
└─────────────────────────────────────────────┘
第三步：海关节点验证
海关节点收到数据包后：


# 受信任节点 A 先验签（不接触敏感数据）
def verify_packet(packet):
    # 1. 重新计算明文部分的 SM3 摘要
    raw_data = {
        "decl_id": packet["decl_id"],
        "company": packet["company"],
        "hs_code": packet["hs_code"],
        ...
    }
    hash = sm3_hash_string(json.dumps(raw_data))
    
    # 2. 用企业的公钥验签
    # SM2 验签：verify(pk_A, hash, signature)
    is_valid = sm2_verify(packet["public_key"], hash, packet["signature"])
    
    return is_valid  # True → 数据未被篡改，身份确认

验证流程：
  数据包签名 ──→ 用 pk_A 验签 ──→ 通过则确认：
    ① 数据确实来自企业 A（身份认证）
    ② 数据在传输中未被篡改（完整性）
    ③ 企业 A 不能抵赖（不可否认性）
只有拥有海关私钥的授权节点才能进一步解密：


# 有海关私钥的节点（如总署节点）解密敏感数据
def decrypt_sensitive(packet):
    value = sm2_decrypt(sk_C, pk_C, packet["encrypted_value"])
    goods = sm2_decrypt(sk_C, pk_C, packet["encrypted_goods"])
    return value, goods
这里就回答了你的上一个问题：解密时明明只需要私钥 sk_C，但 gmssl 库要求传 pk_C，这只是库的接口限制。

第四步：海关生成回执并签名

customs_receipt = {
    "decl_id": "202605120001",
    "status": "approved",              # approved / rejected / pending_inspection
    "inspector": "SH_Customs_001",
    "approved_at": "2026-05-12T10:05:00Z"
}

# ① 海关私钥签名回执
receipt_hash = sm3_hash_string(json.dumps(customs_receipt))
receipt_signature = SM2_Sign(sk_C, receipt_hash)

# ② 可选：用企业公钥加密回执（如果回执本身含敏感信息）
# 但通常回执只有状态信息，可以不加密

receipt_packet = {
    "receipt": customs_receipt,
    "signature": receipt_signature,
    "public_key": pk_C  # 海关公钥
}

第六步：事后验证与审计
任何参与方拿到区块后可以验证：


验证内容                             使用的密钥
─────────────────────────────────────────────
① 区块头 prev_hash 链式完整性         → 计算对比
② 企业提交数据未被篡改               → 企业公钥验签
③ 海关回执真实                       → 海关公钥验签
④ 敏感数据只有海关能看到             → 只有海关私钥能解密
⑤ 区块由合法共识节点生成             → 共识节点公钥验签

关键设计原则总结
原则	实现
最小泄露	只有敏感字段加密，非敏感字段明文可查
分级授权	所有节点可验签，只有海关节点可解密敏感数据
不可否认	企业和海关各自签名，链上留痕
隐私保护	商业数据对非授权节点不可见
可审计	任何节点可验签验证数据完整性
这就是你的 SM2 代码在实际海关报关区块链中的完整用法——公钥像"锁"，谁都可以锁上数据；私钥像"钥匙"，只有持有者能打开。签名则反过来，私钥"盖章"，公钥"验证印章真伪"。
